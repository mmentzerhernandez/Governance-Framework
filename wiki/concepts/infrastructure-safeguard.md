# The Infrastructure Safeguard

> **Package component #5.** Embed a DPI Open-Standards Test into the evaluation criteria for all
> incoming foreign development-finance digital infrastructure proposals.

## What it is
A four-criterion screen — derived from India's [[india-dpi-stack]] — applied to any
development-financed digital/AI infrastructure proposal to prevent vendor lock-in before signature.
See [[vendor-lock-in-dpi-test]] for the full criteria table. Reinforced by requiring DPGA-registry
status ([[undp-dpg-standard]]) and GCF-model conditionality clauses ([[green-climate-fund]]).

## Why no new law is needed
It is an **evaluation criterion inserted into existing procurement/appraisal processes** — a scoring
rule, not a statute. See [[zero-new-legislation-pathway]].

## Transferable mechanism (steps)
1. Add the four-criterion DPI Test (API openness, data portability, interoperability, governance) to
   infrastructure tender evaluation.
2. Flag any proposal failing 2+ criteria for a mandatory lock-in risk review.
3. Require DPGA-registry status or an equivalent open-standards assessment.
4. Insert data-portability + exit-planning clauses (migration docs + APIs within 90 days) — see
   [[world-bank-procurement]].

## Trigger / timing
**Protect phase, Months 12–24** — applied to incoming foreign development-finance proposals and
paired with [[eu-ai-act]] conformity clauses in high-stakes contracts.
